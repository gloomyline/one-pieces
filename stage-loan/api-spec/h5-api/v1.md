# 通用说明

## 通用请求参数
以下参数对于列表类的返回结果可用：
- `offset`: 开始位置。默认0
- `limit`: 返回结果数量

## 通用返回结果字段
- `status`: 访问结果，可能的值包括：
  - `SUCCESS`
  - `FAILURE`
  - `TOKEN_ERROR`: 登录状态Token不正确
  - `PERMISSION_ERROR`: 权限错误
- `error_message`: 访问失败的提示信息
- `has_more`: 如返回结果是一个列表，是否接着还有更多的结果可以请求
- `results`: 返回结果。注意无论返回结果应该是一个列表还是单一元素，这个字段都会是一个列表。

## 获取分期金额明细
```
POST /get-amount
```
【token不需要】

#### 请求参数
- `term`: 借款期数【必填】
- `amount`: 商品总价【必填】，商品总价 = 商品单价 * 数量

#### 结果示例
```
{
	"status": "SUCCESS",
	"error_message": "",
	"results": {
		"total_amount": 13083.42, // 合计应还总额
		"monthly": 1090.29 // 每月应还金额
	}
}
```

## 消费分期借款提交
```
POST /loan-consume-submit
```
【token需要】

#### 请求参数
- `period`: 借款期数【必填】
- `shop_id`: 商户ID【必填】
- `details`: 产品详情【必填】【array】,参数格式参考如下
```
[
    {
        "shop_product_id": 1, // 商品ID,通过shop-product接口获得
        "quantity": 0, // 购买数量
        "spec_id": 1 // 规格ID
    },
    {
        "shop_product_id": 1, // 商品ID,通过shop-product接口获得
        "quantity": 2, // 购买数量
        "spec_id": 2 // 规格ID
    }
]
```
- `sign_pic`: 亲签照【必填】【object】,参数格式参考如下
```
{
  "pic_url": "", // 亲签照图片地址
  "img_b64": "data:image/jpeg;base64,qeJ/9k...", // 【pic_url为空时必填】待上传的图片 base64字符串带形如 data:image/png;base64, 头部
  "suffix": "jpg" //  【pic_url为空时必填】图片后缀
}
```

#### 结果示例
```
{
    "status": "SUCCESS",
    "error_message": "提交申请成功"
}
```
